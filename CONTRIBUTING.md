# Contributing to GHAS License Audit Tool

Thank you for your interest in contributing!

## Reporting Issues

1. Search existing issues to avoid duplicates
2. Create a new issue with:
   - Environment details (PowerShell/Bash version, GitHub CLI version, jq version)
   - Steps to reproduce (for bugs)
   - Expected vs. actual behavior
   - Error messages or logs

## Submitting Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes following code style guidelines
4. Test thoroughly (different org sizes, Enterprise Cloud and non-Enterprise scenarios)
5. Commit: `git commit -m "Add: brief description"`
6. Push and open a Pull Request with clear description and reference to related issues

## Code Style

- Clear, descriptive variable names
- Comments for complex logic
- Consistent indentation (4 spaces)
- Error handling where appropriate
- PowerShell/Bash best practices
